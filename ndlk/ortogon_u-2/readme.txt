Универсальная программа проверки ДЛК порядков [4..36] на марьяжность (на наличие ортогональных диагональных соквадратов).

При запуске запрашивается порядок ДЛК (число от 4 до 36 включительно), проверяемых в текущем сеансе. Также порядок ДЛК можно передать в качестве аргумента командной строки, например: ortogon_u.txt 9, в этом случае ручной ввод порядка будет запрошен только в том случае если через аргумент командной строки было передано некорректное значение.

По умолчанию имя входного файла — input.txt. Если имя другое, то его можно передать как аргумент командной строки (если имя не начинается с цифры или слеша), например: ortogon_u.exe my_input.txt. В первом случае имя выходного файла — output.txt, а во втором оно равно имени входного файла с добавленным префиксом "m_", например: m_my_input.txt.

Предусмотрен ключ /m (или /M) для автоматического выхода из программы по её завершении без необходимости ручного подтверждения, что может быть полезно в пакетных файлах.

Порядок аргументов командной строки и ключей неважен.

Формат входного файла практически произволен. Фактически для ДЛК порядка N из входного файла считываются только N-ричные цифры (первые N символов последовательности [0..9A..Z], где регистр букв неважен) все прочие символы игнорируются. Также входной файл может содержать комментарии — произвольный текст (не содержащий ']'), заключённый в квадратные скобки, например: [Это комментарий]. Комментарии полностью игнорируются.

Для каждого марьяжного ДЛК в выходной файл будет включён следующий текст:

[DLK(m)], здесь m — число диагональных соквадратов
сам ДЛК
[mate#1]
соквадрат 1
...........
[mate#m]
соквадрат m
пустая строка

Например:
[DLK(1)]
1 2 0 3
3 0 2 1
2 1 3 0
0 3 1 2
[mate#1]
0 1 2 3
2 3 0 1
3 2 1 0
1 0 3 2

[DLK(1)]
1 3 0 2
0 2 1 3
2 0 3 1
3 1 2 0
[mate#1]
0 1 2 3
3 2 1 0
1 0 3 2
2 3 0 1
